# 图片真实性检测配置手册

本文说明综测平台如何启用独立的 AI 生成图 / P 图风险检测。当前检测接口只作为辅助风险提示，不接入申报、审核、归档或计分自动决策。

## 1. 检测架构

平台后端保留统一入口：

```text
POST /api/v1/ai-audits/image-authenticity
```

调用链如下：

```text
综测后端
  -> 文件头 / 扩展名 / 数据库元数据一致性检查
  -> c2patool 读取 C2PA / Content Credentials
  -> 外部 GPU 图片鉴别服务
       -> PDF 拆页
       -> SynthID / 供应商水印检测
       -> TruFor 等篡改定位模型
       -> UniversalFakeDetect 类 AI 生成图检测模型
       -> 融合输出风险概率与可疑区域
```

证据权重建议：

- C2PA / Content Credentials：强溯源信号。命中可信凭证可以降低风险，但没有凭证不代表造假。
- SynthID / 供应商水印：强信号但覆盖有限，通常只能识别对应厂商生成或编辑的内容。
- TruFor / 篡改定位：概率证据，适合发现局部 P 图、拼接、替换和异常区域。
- UniversalFakeDetect 类模型：概率证据，适合通用 AI 生成图风险提示。
- 最终结论只能作为人工审核参考，不能自动驳回。

参考项目：

- C2PA Tool: https://opensource.contentauthenticity.org/docs/c2patool/c2patool-index/
- c2patool GitHub: https://github.com/contentauth/c2patool
- Google SynthID: https://deepmind.google/technologies/synthid
- TruFor: https://grip-unina.github.io/TruFor/
- UniversalFakeDetect: https://github.com/Yuheng-Li/UniversalFakeDetect

## 2. 后端 `.env` 配置

`.env` 建议加入：

```env
IMAGE_AUTHENTICITY_PROVIDER=hybrid
IMAGE_AUTHENTICITY_API_URL=http://127.0.0.1:8100/v1/image-authenticity
IMAGE_AUTHENTICITY_API_KEY=replace-with-image-authenticity-key
IMAGE_AUTHENTICITY_MODEL=c2pa+synthid+trufor+universal_fake_detect
IMAGE_AUTHENTICITY_TIMEOUT_SECONDS=30
IMAGE_AUTHENTICITY_C2PATOOL_PATH=c2patool
```

说明：

- `IMAGE_AUTHENTICITY_API_URL` 为空时，只执行本地元数据检查和可选 C2PA 检查。
- `IMAGE_AUTHENTICITY_API_KEY` 配置后，后端会以 `Authorization: Bearer <key>` 调用外部服务。
- `IMAGE_AUTHENTICITY_C2PATOOL_PATH` 可以是 `c2patool`，也可以是完整路径，例如 `C:\tools\c2patool.exe`。
- `IMAGE_AUTHENTICITY_TIMEOUT_SECONDS` 建议 30 到 90 秒。PDF 多页检测较慢时可调大。

## 3. c2patool 安装

### Windows PowerShell

1. 从 `contentauth/c2patool` 或 `contentauth/c2pa-rs` 的 Releases 页面下载 Windows 压缩包。
2. 解压到固定目录，例如：

```powershell
mkdir C:\tools\c2patool
# 将 c2patool.exe 放入 C:\tools\c2patool
setx PATH "$env:PATH;C:\tools\c2patool"
```

3. 新开终端验证：

```powershell
c2patool --version
c2patool .\example.png --json
```

### WSL / Linux

推荐下载官方 release 二进制，放到 `/usr/local/bin`：

```bash
chmod +x c2patool
sudo mv c2patool /usr/local/bin/c2patool
c2patool --version
```

如果使用 `cargo install` 或发行版包管理器，以官方文档说明为准。

## 4. 外部 GPU 检测服务环境

建议把外部鉴别服务做成独立 Python 服务，端口使用 `8100`。不要把重模型直接塞进综测后端进程。

推荐硬件：

- NVIDIA GPU，8GB 显存可跑轻量或单图队列；16GB 以上更适合并发与 PDF 多页。
- CPU 16GB 内存起步。
- Linux / WSL2 / Windows 均可，生产环境推荐 Linux。

推荐 Python 环境：

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -U pip wheel
pip install fastapi "uvicorn[standard]" python-multipart pydantic pillow opencv-python-headless numpy pdf2image
```

PyTorch 需要按服务器驱动和 CUDA 版本选择。示例：

```bash
# 仅示例，生产以 PyTorch 官网安装选择器为准
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu124
```

模型依赖建议：

```bash
pip install timm transformers scikit-image scipy
```

PDF 支持需要 Poppler：

```bash
# Ubuntu / WSL
sudo apt-get update
sudo apt-get install -y poppler-utils

# Windows
# 安装 Poppler for Windows 后，把 bin 目录加入 PATH
```

## 5. 外部服务接口契约

综测后端会以 multipart 形式调用：

```http
POST /v1/image-authenticity
Authorization: Bearer <IMAGE_AUTHENTICITY_API_KEY>
Content-Type: multipart/form-data
```

表单字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `file` | file | 原始证明文件，支持 jpg/png/webp/pdf |
| `file_id` | string | 综测后端文件 ID |
| `filename` | string | 原始文件名 |
| `file_kind` | string | `image` 或 `pdf` |
| `model` | string | 后端配置的模型组合名 |
| `pipeline` | string | `c2pa,synthid,trufor,universal_fake_detect` |

响应 JSON：

```json
{
  "ai_generated_probability": 0.12,
  "tampering_probability": 0.08,
  "watermark": {
    "detected": false,
    "provider": null,
    "confidence": 0.0
  },
  "tampering_regions": [
    {
      "page": 1,
      "bbox": [120, 240, 360, 420],
      "score": 0.91,
      "label": "copy_move_or_splice"
    }
  ],
  "page_results": [
    {
      "page": 1,
      "ai_generated_probability": 0.12,
      "tampering_probability": 0.08,
      "risk_level": "low"
    }
  ],
  "model_versions": {
    "trufor": "trufor-cvpr2023",
    "ai_detector": "universal-fake-detect-calibrated"
  },
  "result": "likely_real"
}
```

建议结果枚举：

- `likely_real`
- `suspected_ai_generated`
- `suspected_tampered`
- `suspected_ai_generated_and_tampered`
- `inconclusive`

PDF 处理要求：

- 外部服务自行拆页。
- 每页独立检测，`page_results` 返回逐页概率。
- `tampering_regions[].page` 必须标记页码。
- 总体 `ai_generated_probability` 和 `tampering_probability` 取最高风险页或融合结果。

## 6. curl 联调

先上传证明文件，拿到 `file_id`，再调用：

```bash
curl -X POST "http://127.0.0.1:8000/api/v1/ai-audits/image-authenticity" \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "file_id": "file_xxxxx",
    "run_c2pa": true,
    "run_external": true
  }'
```

如果只想验证本地 C2PA：

```bash
curl -X POST "http://127.0.0.1:8000/api/v1/ai-audits/image-authenticity" \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "file_id": "file_xxxxx",
    "run_c2pa": true,
    "run_external": false
  }'
```

## 7. 可靠性与审核建议

当前技术不能 100% 判断一张图是否 AI 生成或 P 图，原因包括：

- 平台转存、截图、PDF 拼接、二次压缩会破坏水印、元数据和模型特征。
- 真实奖状扫描件可能因低清晰度、印章覆盖、裁剪而触发误报。
- AI 图检测模型经常只对训练过的生成器表现较好，对新模型会退化。
- 篡改定位模型输出的是可疑区域，不等于法律或纪律意义上的造假证明。

推荐审核口径：

- 高风险只进入人工复核，不自动驳回。
- 人工复核时结合 OCR、姓名匹配、奖项官网、公示名单、学院活动记录一起判断。
- 保留外部服务原始 `raw`、模型版本和阈值配置，便于后续追溯。

## 8. 常见问题

### 返回 `external_ensemble not_configured`

说明没有配置 `IMAGE_AUTHENTICITY_API_URL`，这是正常降级。配置外部服务后重启 API。

### 返回 `c2patool unavailable`

说明系统 PATH 找不到 `c2patool`。安装后设置：

```env
IMAGE_AUTHENTICITY_C2PATOOL_PATH=/usr/local/bin/c2patool
```

Windows 可设置：

```env
IMAGE_AUTHENTICITY_C2PATOOL_PATH=C:\tools\c2patool\c2patool.exe
```

### GPU 不可用

先检查：

```bash
nvidia-smi
python -c "import torch; print(torch.cuda.is_available())"
```

如果 GPU 不可用，外部服务可临时切 CPU，但建议降低并发并延长超时。

### PDF 检测很慢

建议外部服务限制最大页数、缩放到固定分辨率、启用队列，并把 Celery worker 与检测服务分开部署。
