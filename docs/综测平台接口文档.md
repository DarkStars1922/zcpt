# 综测平台接口文档

[TOC]

## 1. 基础约定

### 1.1 Base URL

```text
/api/v1
```

### 1.2 统一响应

除文件流接口外，统一返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "request_id": "uuid"
}
```

### 1.3 鉴权

需要登录的接口统一携带：

```http
Authorization: Bearer <access_token>
```

### 1.4 说明

- 下文“返回示例”只展示 `data` 字段。
- 文件下载类接口返回二进制文件流，不返回 JSON。
- 分数官方来源为 `student_score_summary`。申报的 `item_score / total_score` 是单条申报分，学生汇总分、封顶、超额分和最终分统一由分数服务重算。
- 当前计分只统计当前秋季填报学期创建的申报。学期口径使用 `2025-2026-1`，展示含义为“2025-2026 学年秋季填报学期”；系统默认只使用 `-1`，旧填报周期数据不进入当前总分。
- 申报 `category` 仅允许 `physical_mental`、`art`、`labor`、`innovation`；`sub_type` 仅允许 `basic`、`achievement`。创新素养的 `achievement` 展示为“突破提升”。
- 教师或管理员审核通过时即入账；归档不重复计分；申诉通过后如取消或调整成果，会立即重算并扣回/补录分数。
- 角色说明：
  - `student`：普通学生
  - `reviewer`：已激活审核员令牌的学生
  - `teacher`：教师 / 辅导员
  - `admin`：管理员

## 2. 认证接口

### POST /auth/register

- 请求类型：`POST`
- 鉴权：否
- 角色：未登录用户
- 数据范围：创建一个新用户账号；当前实现允许提交 `student`、`teacher`、`admin` 三种角色值。
- 请求体：

```json
{
  "account": "20230001",
  "password": "Passw0rd!",
  "name": "张三",
  "role": "student",
  "class_id": 301,
  "is_reviewer": false,
  "email": "zhangsan@example.com",
  "phone": "13800000000"
}
```

- 返回示例：

```json
{
  "user": {
    "id": 1,
    "account": "20230001",
    "name": "张三",
    "role": "student",
    "class_id": 301,
    "is_reviewer": false,
    "reviewer_token_id": null,
    "email": "zhangsan@example.com",
    "phone": "13800000000"
  }
}
```

### POST /auth/login

- 请求类型：`POST`
- 鉴权：否
- 角色：已注册用户
- 数据范围：仅返回当前登录用户自己的身份信息与令牌。
- 请求体：

```json
{
  "account": "20230001",
  "password": "Passw0rd!"
}
```

- 返回示例：

```json
{
  "user": {
    "id": 1,
    "account": "20230001",
    "name": "张三",
    "role": "student",
    "class_id": 301,
    "is_reviewer": false,
    "reviewer_token_id": null,
    "email": "zhangsan@example.com",
    "phone": "13800000000"
  },
  "access_token": "eyJhbGciOi...",
  "refresh_token": "eyJhbGciOi...",
  "expires_in": 7200
}
```

### POST /auth/refresh

- 请求类型：`POST`
- 鉴权：否
- 角色：持有有效刷新令牌的用户
- 数据范围：仅刷新当前刷新令牌对应用户的访问令牌。
- 请求体：

```json
{
  "refresh_token": "eyJhbGciOi..."
}
```

- 返回示例：

```json
{
  "access_token": "eyJhbGciOi...",
  "expires_in": 7200
}
```

### POST /auth/logout

- 请求类型：`POST`
- 鉴权：是
- 角色：`student`、`teacher`、`admin`
- 数据范围：仅注销当前访问令牌和提交的刷新令牌，不影响其他用户。
- 请求体：

```json
{
  "refresh_token": "eyJhbGciOi..."
}
```

- 返回示例：

```json
{}
```

### POST /auth/change-password

- 请求类型：`POST`
- 鉴权：是
- 角色：`student`、`teacher`、`admin`
- 数据范围：仅修改当前登录用户自己的密码；修改后当前访问令牌会失效，其他刷新令牌会被撤销。
- 请求体：

```json
{
  "old_password": "Passw0rd!",
  "new_password": "NewPassw0rd!"
}
```

- 返回示例：

```json
{}
```

## 4. 文件接口

### POST /files/upload

- 请求类型：`POST`
- 鉴权：是
- 角色：`student`、`teacher`、`admin`
- 数据范围：上传后的文件归当前上传者所有；后续附件绑定和下载都基于该归属判断。
- 请求体：`multipart/form-data`，字段名为 `file`
- 支持类型：`pdf`、`jpg`、`jpeg`、`png`、`webp`
- 行为说明：
  - 文件保存成功后会立即写入 `queued` 文件分析记录，并通过 Celery 异步执行 OCR 文件分析。
  - 上传本身不等待 OCR 完成；分析结果稍后可通过 `GET /files/{file_id}` 的 JSON 模式查看。
  - 即使分析失败，上传本身仍返回成功，失败信息体现在 `analysis_status` 与 `analysis.error_message`。
- 返回示例：

```json
{
  "file_id": "file_9f14aa78.pdf",
  "filename": "proof.pdf",
  "content_type": "application/pdf",
  "size": 245760,
  "url": "/api/v1/files/file_9f14aa78.pdf",
  "created_at": "2026-04-03T10:00:00+00:00",
  "analysis_status": "completed",
  "analysis": {
    "status": "completed",
    "provider": "paddleocr",

      "省级",
      "一等奖"
    ],
    "uploader_name_match": {
      "expected_name": "张三",
      "matched": true,
      "matches": [
        "张三"
      ]
    },
    "filename_vs_document_title": {
      "filename": "proof.pdf",
      "document_title": "省级创新竞赛一等奖证书",
      "matched": false,
```

### GET /files/{file_id}/url

- 请求类型：`GET`
- 鉴权：是
- 角色：`student`、`reviewer`、`teacher`、`admin`
- 数据范围：与 `GET /files/{file_id}` 一致，先校验访问权限，再返回可访问 URL。
- 返回示例：

```json
{
  "url": "/api/v1/files/file_9f14aa78.pdf"
```

- 请求类型：`DELETE`
- 鉴权：是
- 角色：`student`、`teacher`、`admin`
- 数据范围：仅上传者本人可删除自己的文件；管理员也可删除。
- 返回示例：

```json
{}
```

- 鉴权：是
- 角色：`student`
- 数据范围：只能创建当前登录学生自己的申报；`attachments` 只能引用本人上传的文件。

```json

      "file_id": "file_9f14aa78.pdf"
    }
  ],
  "category": "innovation",
  "sub_type": "achievement",
  "score": 8
}
```

```json
{
  "id": 101,
  "application_id": 101,
  "status": "pending_ai",
```

## 6. AI 审核接口

### GET /ai-audits/{application_id}/report

- 请求类型：`GET`
- 鉴权：是
- 角色：`student`、`teacher`、`admin`
- 数据范围：学生只能查看自己的 AI 审核报告；教师和管理员可查看全部申报的 AI 审核报告。
- 说明：
  - `provider` 默认是 `paddleocr`
  - `identity_check` 用于展示姓名命中情况
  - `consistency_check` 用于展示标题、级别、文件名、印章、落款等一致性校验结果
- 返回示例：

